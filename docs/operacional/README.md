Documentation automatically generated by E2E tests (Playwright) and maintained in the repository.

## How It Works

- Test: `backend/tests/e2e/test_guia_completo.py`
- Command: `pytest tests/e2e/test_guia_completo.py --generate-docs -v`
- Output: files in `docs/operacional/` (`STEP_BY_STEP_GUIDE.md`) and `docs/operacional/screenshots/`

## Files

- [Visual Documentation](DOCUMENTACAO_VISUAL.md)
- [Documentation Summary](RESUMO_DOCUMENTACAO.md)
- [STEP_BY_STEP_GUIDE.md](STEP_BY_STEP_GUIDE.md) (automatically generated)
- [screenshots/](screenshots/) (generated screenshots)

## Notes

- Generation is automatic on pushes to `main` (via workflow) and can be triggered manually.
- Users don't need to generate documentation; we keep it updated via CI.
